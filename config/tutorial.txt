The file `config.sumocfg` is the main configuration file used to run the SUMO simulation. To start the simulation, first navigate to the directory containing the configuration file using `cd /d "D:\Copy\OneDrive\Desktop\my_sumo_project\config"`, and then run the command `sumo-gui -c config.sumocfg`. The file `111.net.xml` is the road network file that represents the map of the study area. The file `route_multitype.rou.xml` is the vehicle configuration file, which defines different vehicle types such as cars, trucks, and motorcycles, as well as parameters like speed limits. The file `config.sumocfg` serves as the main configuration file that specifies the paths for loading both the road network and the vehicle route configuration files. In this project, improvements were made to implement dynamic simulation. Based on travel-time-induced congestion levels, a dynamic route file `dynamic_route_multitype.rou.xml` was generated for the simulation, while `route_multitype.rou.xml` represents the static routing configuration. Additionally, in order to retain the vehicle heading angle during the SUMO simulation for turning-conflict detection, the simulation command was modified to `sumo -c config.sumocfg --fcd-output vehicle_trajectories.xml`.
